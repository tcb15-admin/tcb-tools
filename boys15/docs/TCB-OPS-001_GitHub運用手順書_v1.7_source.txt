3. 日常の更新フロー

3-1 マスタデータの更新（最重要）
過去担当回数（PAST）は、ツール画面上の「この内容で実施確定（PAST に反映）」実行後にのみ更新されます。「割振りを実行」だけでは PAST は変わりません。
※ Cloudflare 同期を有効にしている場合：「保存」で Worker（D1）に保存され、他端末は読込時にクラウドから取得します。GitHub への master.json アップロードは必須ではありません（バックアップ用にダウンロードする場合のみ従来手順）。
※ 同期を使わない端末向けに master.json を GitHub に載せる運用にする場合のみ、以下手順2〜5を行ってください。
手順	操作
1	活動日で割振りを実行し、必要なら担当を手直ししたうえで「この内容で実施確定（PAST に反映）」を実行（またはマスタ設定を変更）
2	「保存」をタップ → 同期 ON ならクラウドへ保存。同期 OFF なら master.json が自動ダウンロード
3	（同期 OFF のみ）GitHub の boys15/ を開き「Add file」→「Upload files」
4	（同期 OFF のみ）master.json をアップロード → 「Commit changes」
5	全端末が次回アクセス時に自動で最新マスタを取得（同期 ON はクラウド、OFF は URL または端末保存）

3-2 共通機能の改修（テンプレート更新）
ツールの機能改修は tool_template.html（および必要なら tcb-print-pdf.js・tcb-sync-api.js）を更新し、全世代をビルドしてください。
手順	操作
1	修正 → tool_template.html（必要なら tcb-print-pdf.js / tcb-sync-api.js）を更新
2	GitHub の template/ に反映（push）
3	template/config_boys15.json および config_boys16.json の TOOL_VERSION を更新（例：1.6.2→1.6.3）
4	（推奨）環境変数 SYNC_API_TOKEN を設定して build.py を実行。未設定時は config 内プレースホルダのまま
5	リポジトリルートで python3 template/build.py → boys15/index.html・boys16/index.html・各 tcb-print-pdf.js・tcb-sync-api.js が生成される
6	boys15/ と boys16/ に、index.html・tcb-print-pdf.js・tcb-sync-api.js を同じコミットで push（片方だけだと同期・PDF が壊れる可能性あり）

3-3 世代固有の設定変更
template/config_boys15.json または config_boys16.json を編集（UI_SIMPLE・SYNC_API_BASE_URL 等）
build.py を実行し、当該世代の index.html（と js）を GitHub にアップロード

3-4 16期の初期マスタ埋め込みを変える場合
template/master_defaults_boys16.json を編集し build.py を再実行（boys16/index.html 再生成）

3-5 道具写真の追加
boys15/images/ または boys16/images/ に画像をアップロード（フォルダが無ければ作成）
マスタ設定 → 道具の説明・写真 → 画像 URL 欄に例として以下を入力：
https://raw.githubusercontent.com/tcb15-admin/tcb-tools/main/boys15/images/【ファイル名】
「保存」→ 同期 ON ならクラウドへ。OFF なら master.json を GitHub にアップロード

3-6 16期マニュアル PDF の更新
boys16/docs/TCB-MAN-016_道具割り振りツール_操作マニュアル_v1.5.md を編集
cd boys16/docs/scripts && node build-pdf.mjs で PDF 再生成
同じファイル名（…v1.5.pdf）で push し URL を変えない（ブックマーク維持）

6. トラブルシューティング
症状	対処方法
URL にアクセスできない	GitHub Pages 有効化後 5〜10 分待つ。Settings → Pages で main・/(root) が Save 済みか。リポジトリが Public か確認
最新マスタが反映されない	同期 ON なら「保存」で Cloudflare を確認。同期 OFF なら master.json の GitHub 反映を確認。スマホは再読み込み
PAST が想定と違う	「実施確定」を押した日付・担当が最終状態か確認。試行の「割振りを実行」だけでは PAST は増えない
持ち帰り調整で担当が消えた	前当番が基準にいない道具は引き継ぎ0件になりうる。STEP3 で手動割当。固定担当は維持される
PDF が出ない・共有できない	index.html と同じフォルダに tcb-print-pdf.js・jspdf.umd.min.js・html2canvas.min.js・html2pdf.min.js があるか確認（v1.7.4 から個別同梱。無い場合は CDN フォールバックに頼るため通信環境の影響を受ける）
パスワードを忘れた	マスタ設定の JSON をエクスポートして管理者に相談
写真が表示されない	raw.githubusercontent.com の URL が正しいか確認
HTML を差し替えたが変わらない	ブラウザのキャッシュをクリア
16期ツールが動かない	SYNC_API_BASE_URL・トークン設定を確認。index.html と tcb-sync-api.js が同じコミットか。ブラウザの開発者ツールのネットワーク／コンソールエラーを確認
