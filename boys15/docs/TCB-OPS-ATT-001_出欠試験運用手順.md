# 15期 出欠アプリ — 試験運用手順

| 項目 | 内容 |
|------|------|
| 文書 | TCB-OPS-ATT-001 |
| 対象 | 東海中央ボーイズ 15期 |
| 目的 | MG LINE／親父 LINE 踏襲の出欠を、少人数で試験して問題を洗い出す |
| 前提 | 道具ツールと同じ Cloudflare 同期・GitHub Pages |

---

## 1. 試験のゴール（合格ライン）

次がすべてできれば試験 OK とする。

1. 出欠マネージャーが **複数日キャンペーン**を作れる  
2. **MG用／親父用**の回答 URL を発行し、各 LINE に案内を貼れる  
3. 保護者（ひとり親を含む）がスマホで回答し、**現行に近い投稿文をコピー**できる  
4. スタッフ画面で **回答済／未回答**が見える  
5. **催促文**を未回答者つきでコピーできる  
6. **受付終了**後、保護者が新規回答できない

## 3. 管理者：デプロイ手順（初回のみ）

Cloudflare にログインできる担当が実施。

### 3.1 D1 マイグレーション

リポジトリの `cloudflare-sync` で:

```bash
cd cloudflare-sync
npx wrangler d1 execute tcb-tools-sync --remote --file=./migrate_attendance.sql
```

※ 旧・単純出欠テーブルがある場合は置き換え（DROP）されます。試験前なら問題ありません。

### 3.2 Worker デプロイ

```bash
cd cloudflare-sync
npm install
npx wrangler deploy
```

### 3.3 フロント再ビルド（トークン注入）

リポジトリルートで（トークンは Git に載せない）:

```bash
SYNC_API_TOKEN='（本番と同じトークン）' python3 template/build.py boys15
```

生成物を GitHub に push し、Pages を更新。

確認ポイント:

- `boys15/attendance/index.html` の `apiToken` がプレースホルダ `__SYNC_API_TOKEN__` のままでないこと  
- `boys15/attendance/kaito.html` にトークン文字列が **含まれていない**こと

## 4. 試験シナリオ（推奨 30〜40 分）

参加者例: 出欠 MGR 1名、MG側回答 2家庭、親父側回答 2家庭（可能ならひとり親想定を1件含む）。

### A. スタッフ準備

1. ポータル → **出欠管理** を開く  
2. ログイン  
3. タイトル例: `【試験】○月○日・○日出欠`  
4. 日付を2日分入れて **作成**  
5. **MG／親父URLを発行**  
6. **MG案内をコピー** → 試験用 MG グループ（または自分宛）へ貼付  
7. **親父案内をコピー** → 試験用親父グループへ貼付  

### B. MG（詳細）回答

1. 案内の URL を開く（画面タイトルが「MG LINE（詳細） 回答」）  
2. 選手を選ぶ  
3. 1日目: 出席連絡で①〜⑦を入力（車種など適当で可）。ひとり親想定なら片側を「― なし」  
4. 2日目: 「休み」＋理由  
5. **送信する** → 表示された投稿文をコピーし、グループへ投稿（試験では自分メモでも可）  
6. もう1家庭でも同様  

### C. 親父（簡易）回答

1. 親父用 URL を開く  
2. 選手を選び、続柄は任意（空欄可）、各日 ◯／△／✕  
3. 送信 → 投稿文をコピー  
4. ※本番運用では LINEスケジュールも従来どおり、と案内に書いてあることを確認  

### D. スタッフ確認

1. 出欠画面で **再読込**  
2. MG／親父の回答数が増えていること  
3. 未回答者が一覧で「未」になっていること  
4. **MG催促をコピー**／**親父催促をコピー**で未回答名が入ること  
5. **受付を終了する** → 保護者 URL を開き、送信できないこと  
6. **受付を再開する**で戻ること  

### E. 退場条件（試験打ち切り）

次があれば Issue／メモして止めてよい。

- 回答が保存されない・別人に付く  
- MG／親父の URL が入れ替わる  
- 投稿文が現行と大きく違い使えない  
- マスタにいない名前だけ出る／対象外（休部等）が出る  

### UX確認（v1.8.3〜）

- [ ] 選手未選択では入力欄が操作できない  
- [ ] 送信中はボタンが「送信中…」になり連打できない  
- [ ] 送信後、投稿文まで自動スクロールする  
- [ ] コピー失敗時（LINE内ブラウザ等）でも文面を選択できる／共有できる  
- [ ] 2回目以降、車種・人数などが覚えられている（MG）  
- [ ] iPhoneで入力欄フォーカス時に画面が過剰ズームしない

## 5. 試験中の運用ルール（本番と分ける）

| 項目 | 試験 |
|------|------|
| LINE 投稿先 | 可能なら試験用トーク、または MGR 限定。本番グループへは「試験」と明記したときだけ |
| データ | 試験キャンペーンのタイトルに【試験】を付ける |
| 従来の出欠 | 試験期間中も本番は従来どおり。切替日は別途決める |
| フィードバック |  conservations: わかりにくい項目・足りない項目・文言の違和感 |

```

※当面、LINEスケジュールへの回答も従来どおりでお願いします。

```
