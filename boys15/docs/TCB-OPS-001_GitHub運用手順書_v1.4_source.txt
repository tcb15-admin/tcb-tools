3. 日常の更新フロー

3-1 マスタデータの更新（最重要）
過去担当回数（PAST）は、ツール画面上の「この内容で実施確定（PAST に反映）」実行後にのみ更新されます。「割振りを実行」だけでは PAST は変わりません。確定後は従来どおり「保存」で master.json をダウンロードし、GitHub にアップロードして全端末へ同期してください。
手順	操作
1	活動日で割振りを実行し、必要なら担当を手直ししたうえで「この内容で実施確定（PAST に反映）」を実行（またはマスタ設定を変更）
2	「保存」をタップ → master.json が自動ダウンロード
3	GitHub の boys15/ または boys16/ を開き「Add file」→「Upload files」
4	master.json をアップロード → 「Commit changes」
5	全端末が次回アクセス時に自動で最新マスタを取得

3-2 共通機能の改修（テンプレート更新）
ツールの機能改修は tool_template.html（および必要なら tcb-print-pdf.js）を更新し、全世代をビルドしてください。
手順	操作
1	Claude 等に修正依頼 → tool_template.html（必要なら tcb-print-pdf.js）を更新
2	GitHub の template/ にアップロード
3	template/config_boys15.json および config_boys16.json の TOOL_VERSION を更新（例：1.4→1.5）
4	build.py を実行 → boys15/index.html・boys16/index.html・各 tcb-print-pdf.js が生成される
5	boys15/ と boys16/ に、index.html と tcb-print-pdf.js の両方をアップロード（片方だけだと PDF 共有等が壊れる可能性あり）

3-3 世代固有の設定変更
template/config_boys15.json または config_boys16.json を編集
build.py を実行し、当該世代の index.html（と tcb-print-pdf.js）を GitHub にアップロード

3-4 16期の初期マスタ埋め込みを変える場合
template/master_defaults_boys16.json を編集し build.py を再実行（boys16/index.html 再生成）

3-5 道具写真の追加
boys15/images/ または boys16/images/ に画像をアップロード（フォルダが無ければ作成）
マスタ設定 → 道具の説明・写真 → 画像 URL 欄に例として以下を入力：
https://raw.githubusercontent.com/tcb15-admin/tcb-tools/main/boys15/images/【ファイル名】
「保存」→ master.json を GitHub にアップロード

6. トラブルシューティング
症状	対処方法
URL にアクセスできない	GitHub Pages 有効化後 5〜10 分待つ。Public リポジトリか確認
最新マスタが反映されない	master.json を GitHub にアップロードしたか確認。スマホはページを再読み込み
PAST が想定と違う	「実施確定」を押した日付・担当が最終状態か確認。試行の「割振りを実行」だけでは PAST は増えない
PDF が出ない・共有できない	index.html と同じフォルダに tcb-print-pdf.js があるか確認。ブラウザが html2pdf.js（CDN）をブロックしていないか確認
パスワードを忘れた	マスタ設定の JSON をエクスポートして管理者に相談
写真が表示されない	raw.githubusercontent.com の URL が正しいか確認
HTML を差し替えたが変わらない	ブラウザのキャッシュをクリア
16期ツールが動かない	boys16/master.json がリポジトリに存在するか確認
